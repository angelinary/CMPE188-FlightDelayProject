# Data Manifest

This document describes every data file in this project — what it is, where it comes from, and how to obtain or regenerate it.

---

## Directory Structure

```
data/
├── raw/
│   └── Airlines.csv              # Source dataset (committed to git)
└── processed/
    ├── airport_weather.csv       # Airport geo + climate cache (committed to git)
    └── Airlines_enriched.csv     # Enriched dataset (NOT committed — regenerate locally)
```

---

## Files

### `data/raw/Airlines.csv`
| Field | Value |
|---|---|
| **Status** | Committed to git |
| **Size** | ~18 MB |
| **Rows** | 539,383 |
| **Source** | [Kaggle — Airlines Dataset to Predict a Delay](https://www.kaggle.com/datasets/jimschacko/airlines-dataset-to-predict-a-delay) |

The original source dataset. Contains 9 columns: `id`, `Airline`, `Flight`, `AirportFrom`, `AirportTo`, `DayOfWeek`, `Time`, `Length`, `Delay`.

> **Note:** If this file is missing, download it from the Kaggle link above and place it at `data/raw/Airlines.csv`.
> *(A direct download link via Google Drive or similar will be added here once available.)*

### `data/processed/airport_weather.csv`
| Field | Value |
|---|---|
| **Status** | Committed to git |
| **Size** | ~17 KB |
| **Rows** | 293 (one per unique airport) |
| **Source** | [OpenMeteo Climate API](https://open-meteo.com/) — free, no key required |

A cached lookup table mapping IATA airport codes to annual climate averages. Generated once by `notebooks/02_feature_engineering.ipynb` (Section 2) and committed so teammates don't need to re-hit the API.

**Columns:** `iata_code`, `avg_temperature` (°C), `avg_precipitation` (mm/day), `avg_wind_speed` (km/h)

> **To regenerate:** Delete this file and re-run `notebooks/02_feature_engineering.ipynb`. The notebook will re-fetch from OpenMeteo (~293 API calls, ~30–60s).

### `data/processed/Airlines_enriched.csv`
| Field | Value |
|---|---|
| **Status** | **NOT committed** (103 MB — exceeds GitHub's 100 MB limit) |
| **Size** | ~103 MB |
| **Rows** | 539,383 |
| **Source** | Generated locally by `notebooks/02_feature_engineering.ipynb` |

The full dataset enriched with geographic and weather features for both origin and destination airports. Adds 12 columns (`from_lat`, `from_lon`, `from_elevation_ft`, `from_avg_temperature`, `from_avg_precipitation`, `from_avg_wind_speed`, and the same for `to_*`).

> **To generate:** Run all cells in `notebooks/02_feature_engineering.ipynb`. Requires `Airlines.csv` and internet access (for OurAirports + OpenMeteo on first run). The weather cache (`airport_weather.csv`) is already committed so only the merge step runs.
>
> *(A shareable download link — Google Drive or similar — will be added here once available.)*

## Maintenance Notes

- `Airlines_enriched.csv` must be regenerated by each team member locally before running notebooks `04_model_tuning.ipynb` or `05_evaluation.ipynb`.
- Never commit `Airlines_enriched.csv` — it is listed in `.gitignore`.
- `airport_weather.csv` only needs to be regenerated if the list of airports in the dataset changes.
